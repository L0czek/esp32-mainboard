# TMP107 Temperature Sensor Chain Support

## Overview

Add TMP107 daisy-chain temperature sensor support to the test stand controller.
TMP107 sensors are connected via half-duplex UART (U0Tx/U0Rx) with a direction
pin (D0) controlling bus direction. Sensors are auto-discovered at startup and
polled at 10Hz, publishing readings to MQTT via the existing temperature packet
infrastructure.

## Hardware Configuration

- **UART**: U0Tx (GPIO16), U0Rx (GPIO17) at 115200 baud, 8N1
- **Direction pin**: D0 (GPIO23) — HIGH = transmit, LOW = receive
- **Protocol**: TMP107 SMAART wire, up to 32 sensors in a daisy chain
- **Bus topology**: Half-duplex single-wire between ESP and sensor chain

## Architecture

### Shared driver: `src/tmp107.rs`

New module in the shared library. Single struct owning UART TX/RX halves and the
direction output pin.

**Struct:**

```rust
pub struct Tmp107 {
    tx: UartTx<'static, Async>,
    rx: UartRx<'static, Async>,
    dir: Output<'static>,
    sensor_count: u8,
}
```

**Constructor (combines new + init):**

```rust
pub async fn init(
    tx: UartTx<'static, Async>,
    rx: UartRx<'static, Async>,
    dir: Output<'static>,
) -> Result<Self, Tmp107Error>
```

Runs the Address Initialize sequence, discovers all sensors in the chain, stores
the count. Returns error if no sensors found or UART fails.

**Internal protocol primitives (private):**

- `send_command(global, read, address, register)` — sends 0x55 calibration +
  command byte + register pointer, handles direction pin
- `individual_read(address, register) -> Result<u16>` — single device read
- `global_read(max_address, register, out) -> Result<usize>` — all devices
  respond sequentially
- `individual_write(address, register, value) -> Result<()>` — single device write
- `global_write(max_address, register, value) -> Result<()>` — broadcast write

**Command byte format:**

```
bit7 = global (G/nI: 1=global, 0=individual)
bit6 = read   (R/nW: 1=read, 0=write)
bit5 = 0      (C/nA)
bits4-0 = address & 0x1F
```

**Register pointer format:**

```
bits7-4 = register address (P3-P0)
bits3-0 = 0101 (fixed identification pattern)
```

**Public API:**

- `read_temperature(address) -> Result<u16>` — reads one sensor via
  `individual_read`
- `read_all_temperatures(out: &mut [u16]) -> Result<usize>` — reads all
  discovered sensors via `global_read`, returns count
- `sensor_count() -> u8` — getter

**Half-duplex flow per transaction:**

1. Set dir HIGH (TX mode)
2. Write: `[0x55, command_byte, register_pointer]`
3. Flush TX
4. Set dir LOW (RX mode)
5. Read response bytes with timeout
6. Return parsed data

**Error type:**

```rust
pub enum Tmp107Error {
    UartWrite,
    UartRead,
    Timeout,
    NoSensorsFound,
}
```

### Collection task: `src/bin/test_stand_controller/temperature_collection.rs`

Binary-specific module following the `SensorCollectionIo` pattern.

**IO struct:**

```rust
pub struct TemperatureCollectionIo {
    pub uart: UART0<'static>,
    pub tx_pin: U0TxPin,
    pub rx_pin: U0RxPin,
    pub dir_pin: D0Pin,
}
```

**Task function:**

```rust
#[embassy_executor::task]
pub async fn temperature_collection_task(io: TemperatureCollectionIo)
```

1. Configure UART at 115200 baud, 8N1
2. Split into UartTx + UartRx
3. Configure dir_pin as Output (initially LOW)
4. Call `Tmp107::init(tx, rx, dir)` — discovers sensors
5. Loop with `Ticker::every(Duration::from_millis(TEMP_COLLECTION_INTERVAL_MS))`:
   - Record timestamp
   - `driver.read_all_temperatures(&mut buf)`
   - For each sensor: create `TempPacket`, call `publish_temperature_sensor()`
   - On error: log warning, continue

**Timing:** Uses `embassy_time::Ticker` so collection time does not add to the
100ms interval. If collection exceeds the interval, next tick fires immediately.

**Config constant** in `config.rs`:

```rust
pub const TEMP_COLLECTION_INTERVAL_MS: u64 = 100;
```

### Wiring in `main.rs`

```rust
let temp_io = TemperatureCollectionIo {
    uart: peripherals.UART0,
    tx_pin: board.U0Tx,
    rx_pin: board.U0Rx,
    dir_pin: board.D0,
};

spawner
    .spawn(temperature_collection::temperature_collection_task(temp_io))
    .expect("Failed to spawn temperature_collection_task");
```

## Error Handling

- **Init failure**: Task logs error and returns. Rest of system unaffected.
- **Read failure**: Log warning, continue loop. Transient bus glitches tolerated.
- **Queue full**: Log warning, drop packet, continue (same as ADC collection).

## Existing Infrastructure (No Changes Needed)

- `TempPacket` in `mqtt/sensors/temp.rs` — stores sensor_id + timestamp + u16 values
- `publish_temperature_sensor()` in `mqtt/queue.rs` — enqueues to outbound queue
- `OutboundMessage::Temp` routing in `mqtt/client.rs` — publishes to `sensor/temp/{id}`
- `format_temp_topic()` in `mqtt/topics.rs` — builds per-sensor topic string
